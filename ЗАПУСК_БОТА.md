# Документация по запуску VK бота

## Методы работы

Бот поддерживает два метода получения обновлений:

### 1. LongPollAPI (Длительное опрашивание)
- **Как работает**: Бот постоянно опрашивает серверы VK на предмет новых сообщений
- **Плюсы**: 
  - Простой в настройке
  - Не требует публичного URL
  - Хорошо подходит для разработки
- **Минусы**:
  - Более высокая задержка в получении сообщений
  - Более интенсивное использование трафика
  - Менее надежен для production
- **Когда использовать**: локальная разработка, тестирование

### 2. CallbackAPI (Webhook)
- **Как работает**: VK отправляет события на ваш сервер через HTTP callbacks
- **Плюсы**:
  - Мгновенное получение обновлений
  - Минимальная задержка
  - Более эффективно в использовании ресурсов
  - Рекомендуется для production
- **Минусы**:
  - Требует публичного URL и открытого порта
  - Более сложная настройка
  - Требует SSL/HTTPS
- **Когда использовать**: production сервер, высокие требования к скорости отклика

## Способы запуска

### Способ 1: Интерактивный выбор (без переменных окружения)

```bash
python main.py
```

Вы увидите меню:
```
==================================================
ВЫБОР МЕТОДА РАБОТЫ БОТА
==================================================
Доступные методы работы:
1 - LongPollAPI (длительное опрашивание)
2 - CallbackAPI (Webhook)
==================================================

Выберите метод (1 или 2): 
```

Введите `1` для LongPollAPI или `2` для CallbackAPI.

---

### Способ 2: Через переменную окружения (Linux/macOS/PowerShell)

#### LongPollAPI:
```bash
export POLLING_METHOD=longpoll
python main.py
```

#### CallbackAPI:
```bash
export POLLING_METHOD=callback
export BASE_URL=https://your-domain.com
export WEBHOOK_SECRET=your_secret_key
python main.py
```

---

### Способ 3: Через переменную окружения (Windows PowerShell)

#### LongPollAPI:
```powershell
$env:POLLING_METHOD="longpoll"
python main.py
```

#### CallbackAPI:
```powershell
$env:POLLING_METHOD="callback"
$env:BASE_URL="https://your-domain.com"
$env:WEBHOOK_SECRET="your_secret_key"
python main.py
```

---

### Способ 4: Через .env файл

1. Скопируйте и отредактируйте файл `.env.example`:
```bash
cp .env.example .env
```

2. Отредактируйте `.env` с нужными значениями:
```
POLLING_METHOD=callback
BASE_URL=https://your-domain.com
WEBHOOK_HOST=0.0.0.0
WEBHOOK_PORT=8080
WEBHOOK_PATH=/
WEBHOOK_SECRET=your_secret_key
```

3. Установите python-dotenv:
```bash
pip install python-dotenv
```

4. В начало `main.py` добавьте:
```python
from dotenv import load_dotenv
load_dotenv()
```

5. Запустите бота:
```bash
python main.py
```

## Переменные окружения для CallbackAPI

| Переменная | Обязательна | Значение по умолчанию | Описание |
|---|---|---|---|
| `POLLING_METHOD` | Нет | `callback` или интерактивный выбор | Метод работы: `longpoll` или `callback` |
| `BASE_URL` | **Да** при callback | - | Публичный URL вашего сервера (https://example.com) |
| `WEBHOOK_HOST` | Нет | `0.0.0.0` | IP адрес, на котором слушает сервер |
| `WEBHOOK_PORT` | Нет | `8080` | Порт для webhook сервера |
| `WEBHOOK_PATH` | Нет | `/` | Path для webhook (https://example.com/path) |
| `WEBHOOK_SECRET` | Нет | - | Секретный ключ для проверки подлинности запросов |

## Настройка CallbackAPI для production

### 1. Установка SSL сертификата
- Используйте Let's Encrypt (бесплатно)
- Используйте nginx или Apache как reverse proxy

### 2. Настройка nginx (пример):
```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3. Запуск бота:
```bash
POLLING_METHOD=callback \
BASE_URL=https://your-domain.com \
WEBHOOK_SECRET=$(openssl rand -hex 32) \
python main.py
```

### 4. Регистрация webhook в VK
Webhook автоматически регистрируется при запуске благодаря параметру `webhook_settings` в `bot.run_webhook()`.

## Отладка

### LongPollAPI медленно реагирует
- Это нормально, задержка может быть до 25 сек
- Рассмотрите переход на CallbackAPI

### CallbackAPI не получает события
1. Проверьте доступность URL:
```bash
curl -X POST https://your-domain.com/ -d "test"
```

2. Проверьте логи бота и сервера

3. Убедитесь, что webhook зарегистрирован в VK API

4. Проверьте firewall и открытость портов

### Ошибка "BASE_URL не задана"
- Для CallbackAPI обязательно нужна переменная `BASE_URL`
- Используйте LongPollAPI если нет публичного URL

## Совет
Для разработки используйте LongPollAPI (проще), для production используйте CallbackAPI (надежнее и быстрее).
